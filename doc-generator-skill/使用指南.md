# 智能文档生成器使用指南

## 概述

智能文档生成器已经成功创建并测试!这个skill可以根据系统信息自动生成7种标准的软件开发文档。

## 已创建的文件

```
doc-generator-skill/
├── skill.md                           # Skill主文件
├── README.md                          # 使用说明
├── config/
│   ├── document-types.json            # 文档类型配置(7种文档)
│   └── template-structure.json        # 模板结构配置
├── scripts/
│   └── generate-document.py           # 文档生成脚本
└── examples/
    └── camunda-input.json             # Camunda输入数据示例
```

## 支持的文档类型

1. **概要设计说明书 (RD-002)** - 系统总体架构、功能模块、技术选型
2. **数据库设计说明书 (RD-001)** - 数据库表结构、索引设计
3. **详细设计说明书 (RD-003)** - 模块详细设计、类设计、算法设计
4. **接口设计说明书 (RD-004)** - API接口详细说明
5. **产品交互设计说明书 (PM-006)** - 用户交互设计、UI设计
6. **用户操作手册 (RM-002)** - 最终用户使用指南
7. **软件测试报告 (TD-002)** - 测试过程和结果记录

## 使用方法

### 1. 列出支持的文档类型

```bash
cd c:\Users\majq1\CodeBuddy\20260326101449\doc-generator-skill
python scripts/generate-document.py --list-types
```

### 2. 准备输入数据

创建JSON文件,包含以下信息:

**必需字段:**
- `systemName`: 系统名称
- `systemUrl`: 系统地址
- `description`: 系统描述
- `version`: 版本号
- `author`: 编写人

**可选字段:**
- `features`: 功能列表
- `techStack`: 技术栈(对象或字符串)
- `databaseType`: 数据库类型
- `tables`: 数据库表信息数组
- `apiDocs`: API接口文档数组
- `userRoles`: 用户角色数组
- `testResults`: 测试结果数组

**示例:**
```json
{
  "systemName": "订单管理系统",
  "description": "电商订单管理系统",
  "version": "v1.0",
  "author": "开发团队",
  "systemUrl": "http://demo.example.com",
  "testAccount": {
    "username": "admin",
    "password": "admin123"
  },
  "features": [
    "订单创建",
    "订单查询",
    "订单支付",
    "订单取消"
  ],
  "techStack": {
    "语言": "Java",
    "框架": "Spring Boot",
    "数据库": "MySQL"
  }
}
```

### 3. 生成文档

```bash
# 生成概要设计文档
python scripts/generate-document.py --type "概要设计说明书" --input input.json --output output.md

# 生成数据库设计文档
python scripts/generate-document.py --type "数据库设计说明书" --input input.json --output db-design.md

# 生成接口设计文档
python scripts/generate-document.py --type "接口设计说明书" --input input.json --output api-design.md
```

## 高级功能

### 自定义文档模板

1. 修改 `config/template-structure.json` 添加自定义章节
2. 修改 `scripts/generate-document.py` 添加生成逻辑
3. 重新生成文档

### 添加新文档类型

1. 在 `config/document-types.json` 添加新类型
2. 在 `config/template-structure.json` 定义结构
3. 在 `scripts/generate-document.py` 实现生成逻辑

### 批量生成脚本

创建批处理脚本一次性生成所有文档:

```bash
#!/bin/bash
# generate-all-docs.sh

INPUT_FILE="input.json"
OUTPUT_DIR="./output"

python scripts/generate-document.py --type "概要设计说明书" --input $INPUT_FILE -o $OUTPUT_DIR/概要设计.md
python scripts/generate-document.py --type "数据库设计说明书" --input $INPUT_FILE -o $OUTPUT_DIR/数据库设计.md
python scripts/generate-document.py --type "详细设计说明书" --input $INPUT_FILE -o $OUTPUT_DIR/详细设计.md
python scripts/generate-document.py --type "接口设计说明书" --input $INPUT_FILE -o $OUTPUT_DIR/接口设计.md
python scripts/generate-document.py --type "用户操作手册" --input $INPUT_FILE -o $OUTPUT_DIR/用户手册.md
python scripts/generate-document.py --type "软件测试报告" --input $INPUT_FILE -o $OUTPUT_DIR/测试报告.md

echo "所有文档生成完成!"
```

## 常见问题

### Q1: 如何处理中文路径和文件名?

Python脚本已使用UTF-8编码,支持中文路径和文件名。

### Q2: 生成的文档不够详细怎么办?

工具提供基础框架和主要内容,可以:
1. 在输入JSON中提供更多详细信息
2. 人工审核后补充细节
3. 修改模板添加更多章节

### Q3: 如何生成Word格式文档?

当前版本生成Markdown格式,可以:
1. 使用Pandoc转换: `pandoc input.md -o output.docx`
2. 在Word中打开Markdown并另存
3. 使用Markdown转Word工具

### Q4: 支持团队协作吗?

支持!生成的Markdown文件可以:
- 提交到Git仓库进行版本管理
- 使用在线Markdown编辑器协作编辑
- 转换为Word后使用Office协作

## 最佳实践

1. **数据准备**: 尽可能收集完整的系统信息
2. **分步生成**: 先生成基础文档,再逐步完善
3. **版本控制**: 将文档纳入Git管理
4. **持续更新**: 系统变更时及时更新文档
5. **人工审核**: 生成后必须人工审核和补充
