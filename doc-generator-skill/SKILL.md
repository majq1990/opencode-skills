# 智能文档生成器 (Smart Document Generator)

## 概述

智能文档生成器是一个自动化的软件开发文档生成工具,能够根据系统服务地址、功能清单和调用说明,自动生成符合企业标准的技术文档。

## 功能特点

- **多文档类型支持**: 支持7种标准软件开发文档
- **模板化生成**: 基于标准模板自动生成文档
- **智能填充**: 根据系统信息自动填充文档内容
- **格式统一**: 确保所有文档符合企业规范
- **快速生成**: 大幅提升文档编写效率
- **多格式输出**: 同时输出 Markdown、DOCX（Word）、PPTX（PowerPoint）三种格式

## 使用流程

### 输入要求

使用本skill需要提供以下信息:

1. **系统访问地址**: 系统的URL地址(如: http://47.110.57.11:8080/camunda)
2. **功能清单**: 系统提供的主要功能列表
3. **调用说明**: API接口说明文档或接口调用示例
4. **文档类型**: 需要生成的文档类型(可多选)
5. **产品信息**: 产品名称、版本号、项目负责人等基本信息

### 生成步骤

1. 分析系统功能和技术架构
2. 根据文档类型选择对应的模板
3. 收集并整理系统信息
4. 填充模板内容
5. 生成 Markdown 格式文档（中间产物）
6. 转换为 DOCX 格式（Word 文档）
7. 生成 PPTX 格式（PowerPoint 演示文稿）

### 输出成果物

每次文档生成必须输出以下三种格式的成果物：

| 格式 | 文件扩展名 | 用途 | 必须输出 |
|------|------------|------|----------|
| Markdown | .md | 中间产物、版本控制、在线预览 | 是 |
| DOCX | .docx | 正式交付文档、打印归档 | 是 |
| PPTX | .pptx | 项目汇报、演示展示 | 是 |

**输出文件命名规范：**
- Markdown: `{文档类型}_说明书.md`
- DOCX: `{文档类型}说明书.docx`
- PPTX: `{系统名称}-项目汇报.pptx`

**输出路径：** `D:\opencode\file\{YYYY-MM-DD}\`

## 生成规则

### 核心规则（必须严格遵守）

1. **严格遵循模板结构**：使用模板定义的标题、层级、格式，不增删任何模块
2. **严格遵循章节顺序**：必须按照模板的章节编号顺序生成，如：1.概述 → 2.总体设计 → 3.系统设计 → 4.接口设计 → 5.数据库设计 → 6.系统出错处理设计
3. **智能拆解归类**：将用户描述的内容自动拆解、归类到对应章节
4. **语言规范正式**：语言正式、简洁，可直接交付用户，无需二次修改
5. **步骤必须编号**：所有操作步骤必须编号（1. 2. 3.），条理清晰
6. **自动填充默认内容**：没有信息的章节自动生成通用标准内容，不向用户提问

**禁止事项：**
- 禁止添加"文档信息"、"修订历史"等作为独立章节
- 禁止重复章节编号
- 禁止调整章节顺序
- 禁止遗漏任何一级章节

### 内容填充规则

1. **系统概述**: 根据访问地址和功能清单自动生成
2. **架构设计**: 基于技术架构信息生成架构图和组件说明
3. **功能模块**: 根据功能清单生成模块划分
4. **接口设计**: 根据API说明生成接口文档
5. **数据库设计**: 根据数据库表结构生成设计文档

### 格式规范

1. 使用Markdown格式
2. 代码块使用语法高亮
3. 表格使用标准Markdown表格语法
4. 标题层级使用#, ##, ###等

### 内容要求

1. 内容必须完整,涵盖所有必需章节
2. 技术描述准确,符合系统实际情况
3. 语言简洁明了,易于理解
4. 符合企业文档规范

### 默认内容生成规则

当用户未提供特定章节信息时，按以下规则自动生成：

| 章节 | 默认内容生成规则 |
|------|------------------|
| 编写目的 | 生成标准化的文档目的说明 |
| 适用范围 | 生成该文档类型的标准适用范围 |
| 术语定义 | 生成该领域的通用术语列表 |
| 参考资料 | 生成行业通用参考资料 |
| 运行环境 | 生成标准硬件/软件/网络要求 |
| 注意事项 | 生成该文档类型的标准注意事项 |

## 配置选项

### 文档配置

```yaml
document:
  productName: "产品简称"
  version: "v1.0"
  author: "编写人"
  date: "2026-03-26"
```

### 输出配置

```yaml
output:
  formats: ["markdown", "docx", "pptx"]  # 必须同时输出三种格式
  path: "D:/opencode/file/{YYYY-MM-DD}"  # 按日期分目录存储
  markdown:
    enabled: true
    extension: ".md"
  docx:
    enabled: true
    extension: ".docx"
    template: "default"  # 可选: default, formal
  pptx:
    enabled: true
    extension: ".pptx"
    slides:
      - 封面
      - 目录
      - 系统概述
      - 功能模块设计
      - 技术架构设计
      - 数据库设计
      - 接口设计
      - 安全设计
      - 感谢页
```

### 技术依赖

生成 DOCX 和 PPTX 格式需要以下 Python 库：
- `python-docx`: 用于生成 Word 文档
- `python-pptx`: 用于生成 PowerPoint 演示文稿

安装命令：
```bash
pip install python-docx python-pptx
```

## 最佳实践

1. **先分析后生成**: 充分了解系统后再生成文档
2. **逐步完善**: 先生成基础框架,再逐步补充细节
3. **保持更新**: 系统变更后及时更新文档
4. **多人协作**: 支持多人协作编辑文档
5. **版本管理**: 对文档进行版本管理

## 注意事项

1. 确保系统可访问,能够收集到必要信息
2. 功能清单应详细完整
3. API说明应包含请求/响应示例
4. 生成后需要人工审核和修正
5. 定期更新文档以保持与系统同步

## 故障排查

### 常见问题

**Q: 文档生成失败?**  
A: 检查输入信息是否完整,系统地址是否可访问。

**Q: 内容不准确?**  
A: 人工审核并修正生成的内容,确保准确性。

**Q: 格式不符合要求?**  
A: 调整模板配置,修改文档格式规范。

**Q: 目录结构与模板不匹配?**  
A: 检查生成的文档是否严格遵循标准目录结构，确保章节编号和顺序正确。
